### server/include/parser/ 

语句解析模块。该模块采用 **AST（抽象语法树）** 来处理 SQL 语句的原始输入。

**核心设计原则：** 数据与行为分离。AST 节点仅声明物理结构，存放由 Parser 组装的碎片数据，不包含任何业务与运算代码。所有的多态节点由 `std::unique_ptr` 严格管理生命周期以防止内存泄漏。

**执行引擎（ExecutorFactory）的调用逻辑：**
执行引擎接收到 AST 根节点指针，通过 `switch (node->getType())` 捕获节点类型，将其安全向下转型为对应的具体节点类（如 `SelectASTNode*`），随后读取节点内部的数据结构去调用 Storage 查数据。

**以SELECT语句为例，该树大概长这样：**
(树根) SelectASTNode
 │
 ├── targetFields [数组]
 │    ├── 第0个元素 ──> ColumnRefNode (表名:A, 列名:name)
 │    └── 第1个元素 ──> FunctionCallNode (函数名:COUNT)
 │                       └── arguments ──> ColumnRefNode (表名:B, 列名:score)
 │
 ├── fromSource [指针]
 │    └── JoinNode (连接类型: INNER JOIN)
 │         ├── leftTable   ──> TableNode (表名:student, 别名:A)
 │         ├── rightTable  ──> TableNode (表名:score, 别名:B)
 │         └── onCondition ──> BinaryOperatorNode (操作符: "=")
 │                              ├── left  ──> ColumnRefNode (表名:A, 列名:id)
 │                              └── right ──> ColumnRefNode (表名:B, 列名:uid)
 │
 └── whereExpressionTree [指针]
      └── BinaryOperatorNode (操作符: ">")
           ├── left  ──> ColumnRefNode (表名:A, 列名:age)
           └── right ──> LiteralNode (值: 18)
#### server/include/parser/ISQLParser.h

- `virtual std::unique_ptr<ASTNode> parse(const std::string& sql) = 0;`
    
    核心接口：接收纯文本 SQL，返回所有权独占的结构化 AST 根节点智能指针。语法错误返回 `nullptr`。
    

#### server/include/parser/ASTNode.h

定义了语法树的基石与表达式节点家族。

- `enum class SQLType`: 包含 `CREATE_TABLE`, `DROP_TABLE`, `INSERT`, `SELECT`, `UPDATE`, `DELETE`, `UNKNOWN`。
    
- `struct ColumnDefinition`: 专门用于存储建表语句（DDL）中高度扁平化的列定义属性（名称、类型、约束等）。
    
- `class ASTNode`: 绝对基类，提供 `virtual SQLType getType() const = 0;` 以及极其关键的虚析构函数 `virtual ~ASTNode() = default;`。
    
- **表达式子节点**（均返回 `UNKNOWN`，作为 DML 语句的组件存在）：
    
    - `ColumnRefNode`: 字段引用节点（如 `A.name`）。
        
    - `FunctionCallNode`: 聚合函数节点（如 `COUNT(B.score)`），内部包含 `arguments` 指针数组。
        
    - `BinaryOperatorNode`: 二元操作符节点（如 `age > 20`），通过 `left` 和 `right` 多态指针形成递归表达式树。
        
    - `JoinNode`: 多表连接节点，包含左右子表指针及 `onCondition` 连接条件树。
        

#### server/include/parser/CreateTableASTNode.h

- **身份**：`SQLType::CREATE_TABLE`
    
- **核心结构**：包含表名 `tableName` 和极其扁平化的值语义数组 `std::vector<ColumnDefinition> columns`。供 CatalogManager 直接序列化为 `.tdf` 文件，不涉及任何执行期计算。
    

#### server/include/parser/DropTableASTNode.h

- **身份**：`SQLType::DROP_TABLE`
    
- **核心结构**：仅包含 `tableName`。
    

#### server/include/parser/InsertASTNode.h

- **身份**：`SQLType::INSERT`
    
- **核心结构**：包含 `tableName`、插入目标列数组 `std::vector<std::string> columns`，以及插入值数组 `std::vector<std::string> values`。
    

####  server/include/parser/DeleteASTNode.h

- **身份**：`SQLType::DELETE`
    
- **核心结构**：包含 `tableName`。
    
- **条件区**：`std::unique_ptr<ASTNode> whereExpressionTree`。采用唯一多态指针承载递归过滤条件，支持复杂的 `AND/OR` 嵌套逻辑判定。
    

#### server/include/parser/UpdateASTNode.h

- **身份**：`SQLType::UPDATE`
    
- **核心结构**：
    
    - `tableName`
        
    - `std::vector<SetClause> setValues`：封装 `SET 字段=值` 键值对。
        
    - `std::unique_ptr<ASTNode> whereExpressionTree`：承载 WHERE 更新过滤条件的多态指针。
        

#### server/include/parser/SelectASTNode.h

查询语句的全生命周期物理建模，包含六大物理区域。

- **身份**：`SQLType::SELECT`
    
- **核心结构**：
    

|**成员变量**|**类型**|**物理含义与作用**|
|---|---|---|
|`isDistinct`|`bool`|是否开启去重过滤 (`SELECT DISTINCT`)|
|`targetFields`|`std::vector<std::unique_ptr<ASTNode>>`|**查询目标区**：存放列、聚合函数或计算表达式。|
|`fromSource`|`std::unique_ptr<ASTNode>`|**数据来源区**：指向单表节点或 JOIN 关联树的根。|
|`whereExpressionTree`|`std::unique_ptr<ASTNode>`|**过滤条件区**：存放用于逐行真假判断的表达式逻辑树。|
|`groupByFields`|`std::vector<std::unique_ptr<ASTNode>>`|**分组依据区**：定义数据分类汇总的键。|
|`havingExpressionTree`|`std::unique_ptr<ASTNode>`|**二次过滤区**：针对分组统计后的结果进行筛选。|
|`orderByItems`|`std::vector<OrderByItem>`|**排序规则区**：定义结果呈现的先后顺序及方向 (`ASC/DESC`)。|

#### server/include/parser/Token.h

定义词法分析的最小数据载体与身份标识。

- **`enum class TokenType`**：定义系统中所有合法的词汇物理分类。
    
    - **关键字 (Keywords)**：如 `KW_SELECT`, `KW_FROM`, `KW_CREATE` 等系统保留字。
        
    - **标识符 (Identifiers)**：`IDENTIFIER`，用于标记用户自定义的表名、列名。
        
    - **字面量 (Literals)**：`NUMBER_LITERAL` (数字), `STRING_LITERAL` (单引号字符串)。
        
    - **符号 (Symbols)**：如 `SYM_EQ` (=), `SYM_GE` (>=), `SYM_LPAREN` (() 等。
        
    - **特殊标记**：`END_OF_FILE` (终止符), `INVALID` (非法字符或词法错误)。
        
- **`struct Token`**：
    
    - `TokenType type;`：当前词元的物理身份枚举。
        
    - `std::string value;`：从内存中提取出的原始字符串切片。
        
    - `int position;`：记录该单词在原始 SQL 字符串中的绝对起始物理下标（用于后续 Parser 精准定位并抛出语法错误坐标）。
        
    - _应用案例_：对于 SQL 中的 `SELECT`，生成的结构体为 `{type: TokenType::KW_SELECT, value: "SELECT", position: 0}`。

#### server/include/parser/Lexer

- 模块概述

	- 词法分析模块是 SQL 解析引擎的第一道工序。其核心职责是将输入的、无结构的 SQL 字符串（字节流）转化为一组具有物理意义的词元（Token）流。该模块采用**确定性有限状态自动机（DFA）**设计，确保对 SQL 语句的扫描是线性且高效的。

- 类结构定义与成员变量

	- `Lexer` 类封装了状态维护与切词逻辑，其内部核心成员如下：

|**成员变量**|**类型**|**物理含义与作用**|
|---|---|---|
|`sql`|`std::string`|存放待处理的原始 SQL 语句完整镜像。|
|`currentPos`|`int`|**核心游标**。记录当前扫描字符在 `sql` 数组中的绝对下标位置。|
|`tokens`|`std::vector<Token>`|存储切分后的结果集，供后续语法分析器（Parser）读取。|
|`keywordMap`|`static const unordered_map`|存储 SQL 保留字的哈希字典，用于 $O(1)$ 复杂度的关键字定性。|


- 核心函数代码实现逻辑
- `tokenize()`：主控驱动逻辑
	- **职责**：负责协调各个提取函数，完成从字符串到 Token 数组的转换。
	- **实现逻辑**：
		1. **初始化**：调用 `tokens.clear()` 确保结果集干净，重置状态。
		2. **主循环**：建立 `while (currentPos < sql.length())` 循环，直到游标触碰字符串末尾
		3. **清洗路障**：在循环体首行调用 `skipSpace()`，强制清理当前位置前方的所有空格、换行或制表符。
		4. **状态探测与分发**
		    - 获取当前字符 `c = sql[currentPos]`。
		    - **分支路由**：
				- 若 `isalpha(c) || c == '_'`：判定为单词起点，调用 `scanIdentifierOrKeyword()`。
			    - 若 `isdigit(c)`：判定为数字起点，调用 `scanNumber()`。
			    - 若 `c == '\''`：判定为字符串常量起点，调用 `scanString()`。
			    - 否则：调用 `scanSymbol()` 处理符号。
		5. **收尾**：循环结束后，压入一个类型为 `END_OF_FILE` 的 Token 作为结束信号。
    

- `skipSpace()`：空白字符清洗
	- **职责**：消除物理内存中的排版噪音。
	- **实现逻辑**：
		1. 采用 `while` 循环，利用 `isspace(sql[currentPos])` 持续检查当前字节。
		2. 若为真（空格、`\n`、`\t` 等），则执行 `currentPos++`，使游标无视该字节并后移。
		3. 循环终止于第一个有效字符或字符串末尾。
    

- `scanIdentifierOrKeyword()`：标识符与关键字定性
	- **职责**：识别并区分 SQL 指令（如 SELECT）与用户定义名（如 student）。
	- **实现逻辑**：
		1. **快照起点**：记录 `start = currentPos`。
		2. **贪婪拼接**：只要当前字符满足 `isalnum`（字母或数字）或等于 `_`，就将其追加到临时字符串 `word` 中，并同步推进 `currentPos`。
		3. **归一化处理**：通过 `std::transform` 将 `word` 转换为全大写。
		4. **哈希匹配**：在 `keywordMap` 中查找该大写串。
		    - **成功**：返回关键字 Token，类型从字典中获取。
		    - **失败**：返回标识符 Token（`IDENTIFIER`），保留用户输入的原始大小写。
        

- `scanString()`：字面量提取

- **职责**：提取单引号包裹的纯文本数据。
	- **实现逻辑**：
		1. **剥离前界**：执行 `currentPos++` 跳过起始单引号。
		2. **文本吞噬**：只要 `currentPos` 未到底且当前字符不是单引号 `'`，则将其作为数据存入 `buffer`。
		3. **闭合校验**：
		    - 若循环因到达末尾而结束，说明字符串未闭合，返回 `INVALID` 错误 Token。
		    - 若因遇到单引号结束，则执行 `currentPos++` 剥离后界。
		4. **打包返回**：封装为 `STRING_LITERAL` 类型的 Token。
    


- `scanSymbol()`：符号与操作符提取
	- **职责**：识别单字符或双字符运算符。
	- **实现逻辑**：
		1. **预提首符**：读取当前字符并执行 `currentPos++`。
		2. **前向探查（Lookahead）**：
		    - 检查 `currentPos` 后的下一个字符。如果是 `>=`、`<=`、`!=` 或 `<>` 组合，则再次执行 `currentPos++`，将两个字符合并为一个双字符 Token。
		3. **映射匹配**：通过 `switch-case` 对识别出的字符组合进行硬编码映射（如 `,` 映射为 `SYM_COMMA`）。
		4. **兜底**：若遇到不认识的乱码字符，返回 `INVALID` 词法错误 Token。
